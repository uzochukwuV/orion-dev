================================================================================
                        WHAT HAPPENS WHEN USER SIGNS UP
================================================================================

1. User goes to /signup
2. Fills email + password
3. Clicks "Sign Up"
4. Clerk verifies and creates user
5. Clerk webhook fires automatically
6. Backend receives: user.created event
7. Backend creates:
   - ClerkUser document
   - Business document (linked to user)
8. User redirected to /dashboard
9. Dashboard loads their data only

Result: Multi-tenant, isolated data per user

================================================================================
                        API SECURITY CHANGES
================================================================================

BEFORE (Broken):
  GET /api/entities/Lead
  → Returns all leads (business_id='demo')
  → All users see same data ❌

AFTER (Secure):
  GET /api/entities/Lead
  Header: Authorization: Bearer {JWT}
  → Backend verifies token with Clerk
  → Looks up user_id
  → Returns only that user's leads ✅

All API calls now require valid JWT token!
