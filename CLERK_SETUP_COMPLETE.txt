## STEP 1: CREATE CLERK ACCOUNT (5 minutes)

1. Go to https://clerk.com
2. Click "Sign Up"
3. Create account with your email
4. Create a new application:
   - Name: "Orion"
   - Choose: "Email + Password + Social" (optional)
5. You'll see your API keys

## STEP 2: COPY API KEYS (2 minutes)

From Clerk Dashboard → API Keys:

**Frontend Key:**
```
REACT_APP_CLERK_PUBLISHABLE_KEY = pk_test_xxxxxxxxxxxxx
```

**Backend Key:**
```
CLERK_SECRET_KEY = sk_test_xxxxxxxxxxxxx
```

**Webhook Secret:**
```
CLERK_WEBHOOK_SECRET = whsec_xxxxxxxxxxxxx
```

## STEP 3: SET UP ENVIRONMENT VARIABLES (3 minutes)

**Frontend (.env or .env.local):**
```
VITE_CLERK_PUBLISHABLE_KEY=pk_test_xxxxxxxxxxxxx
```

**Backend (.env):**
```
CLERK_SECRET_KEY=sk_test_xxxxxxxxxxxxx
CLERK_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxx
```

## STEP 4: SET UP WEBHOOK (5 minutes)

In Clerk Dashboard → Webhooks:

1. Click "Create Endpoint"
2. URL: `http://localhost:3001/api/webhooks/clerk`
3. Events to subscribe to:
   - ✅ user.created
   - ✅ user.updated
   - ✅ user.deleted
4. Copy "Signing Secret" → paste into `CLERK_WEBHOOK_SECRET`

================================================================================
                        WHAT HAPPENS WHEN...
================================================================================

USER SIGNS UP:
  1. User fills email + password in /signup
  2. Clerk creates user in their system
  3. Clerk webhook fires: POST /api/webhooks/clerk
  4. Backend creates ClerkUser + Business in MongoDB
  5. User redirected to /dashboard

USER LOGS IN:
  1. User enters email + password in /login
  2. Clerk verifies credentials
  3. Returns JWT token to browser
  4. Frontend stores token in memory
  5. User sees /dashboard

USER ACCESSES DASHBOARD:
  1. React checks isSignedIn via Clerk
  2. If signed in: show dashboard
  3. If not: redirect to /login
  4. All API calls include JWT token

API CALLS:
  1. Frontend: `Authorization: Bearer {JWT}`
  2. Backend: Verifies token with Clerk
  3. Backend: Looks up ClerkUser → user_id
  4. Backend: Returns data filtered by user_id
  5. Each user sees only their own data

================================================================================
                        COMMON ISSUES
================================================================================

ISSUE: "publishableKey is required"
FIX:
  1. Check VITE_CLERK_PUBLISHABLE_KEY in .env
  2. Restart: npm run dev
  3. Check browser console: should see key loading

ISSUE: Login page is blank
FIX:
  1. Check ClerkProvider is in main.jsx
  2. Check API key is correct in .env
  3. Check internet connection (Clerk CDN)

ISSUE: "Invalid token" error on API calls
FIX:
  1. Check CLERK_SECRET_KEY in backend .env
  2. Restart backend: npm run dev
  3. Verify token is being sent in Authorization header

ISSUE: Webhook not firing
FIX:
  1. Check webhook URL in Clerk Dashboard
  2. URL should be: http://localhost:3001/api/webhooks/clerk
  3. Check webhook secret in CLERK_WEBHOOK_SECRET
  4. Check backend console for errors

ISSUE: "User not found" when accessing API
FIX:
  1. Check MongoDB: Does ClerkUser exist?
  2. Is webhook running? (Check backend logs)
  3. Wait a few seconds for webhook to fire

ISSUE: Still seeing "demo" business data
FIX:
  1. Clear browser cache
  2. Restart backend
  3. Check Business model has user_id filtering
  4. Verify user is logged in
