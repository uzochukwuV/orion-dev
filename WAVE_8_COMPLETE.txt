================================================================================
                        AGENT REST ENDPOINTS
================================================================================

POST /api/agents/run
  Purpose: Run SuperAgent orchestrator
  Input:
    - task (required): User request
    - business_id: Default "demo"
    - skip_confirmation: Auto-approve if true
  Output:
    - JSON: Full result
    - SSE: Real-time step progress + complete event
  Features:
    ✅ Intent classification
    ✅ Agent chain routing + execution
    ✅ Step tracking (onStep callbacks)
    ✅ Approval gates (onConfirm callbacks)
    ✅ Record creation + ID tracking

POST /api/agents/approve
  Purpose: Resume workflow after user approval
  Input:
    - agent_task_id (required): AgentTask MongoDB ID
  Output:
    - { agentTaskId, status: "approved", message }
  Features:
    ✅ Approval state validation
    ✅ Step recording

POST /api/agents/reject
  Purpose: Reject action plan (no record creation)
  Input:
    - agent_task_id (required)
    - reason: Optional rejection reason
  Output:
    - { agentTaskId, status: "rejected", reason }
  Features:
    ✅ Rejection tracking
    ✅ No side effects

POST /api/agents/chat
  Purpose: Direct LLM chat with session history
  Input:
    - message (required): User query
    - session_id: Optional (resume conversation)
    - business_id: Default "demo"
  Output:
    - { reply, session_id, messages: [{role, content}] }
  Features:
    ✅ Session persistence (MongoDB ChatSession)
    ✅ Multi-turn conversations
    ✅ No agents (direct LLM)

GET /api/agents/runs
  Purpose: List recent agent workflows
  Query:
    - business_id: Filter by business
    - limit: Default 20
    - offset: Default 0
    - status: Filter by status
  Output:
    - [{ agentTaskId, task, intent, status, created_at, ... }]
  Features:
    ✅ Sorting by creation time
    ✅ Pagination
    ✅ Status filtering

================================================================================
                    SPEECHMATICS WEBSOCKET RELAY
================================================================================

WS /ws/voice?session=USER_ID&business=BUSINESS_ID
  Purpose: Real-time speech-to-text bridge
  Connection:
    ws://localhost:3001/ws/voice?session=user123&business=demo
  
  Flow:
    1. Browser connects WebSocket
    2. Browser sends audio blobs (binary)
    3. Server relays to Speechmatics ASR
    4. Server sends transcripts back as JSON
    5. Browser displays real-time transcription

  Message Types:
    - { type: "connected", text: "Voice connection established" }
    - { type: "partial", text: "...", isFinal: false, confidence: 0.92 }
    - { type: "final", text: "...", isFinal: true }
    - { type: "error", error: "..." }

  Features:
    ✅ Real-time streaming (100ms chunks)
    ✅ Partial transcripts (mid-utterance)
    ✅ Confidence scores
    ✅ Error recovery
    ✅ Session tracking
    ✅ Business context passing
    ✅ Automatic cleanup on disconnect

  Browser Integration:
    ```javascript
    const ws = new WebSocket('ws://localhost:3001/ws/voice?session=user1');
    
    // Send audio
    navigator.mediaDevices.getUserMedia({audio:true}).then(stream => {
      const rec = new MediaRecorder(stream);
      rec.ondataavailable = e => ws.send(e.data);
      rec.start(100);
    });
    
    // Receive transcripts
    ws.onmessage = e => {
      const msg = JSON.parse(e.data);
      if (msg.type === 'final') {
        // Send to agent or process
      }
    };
    ```

                        DATABASE INTEGRATION
================================================================================

MODELS USED:
  - AgentTaskModel        (workflow state + steps)
  - ChatSessionModel      (conversation history)
  - AgentRunModel         (intelligence scans)
  - OpportunityModel      (research findings)
  - BusinessModel         (context)

READS:
  - BusinessModel.findById()            → context
  - ChatSessionModel.findById()         → session history
  - AgentTaskModel.find()               → workflow list
  - AgentRunModel.find()                → scan history

WRITES:
  - AgentTaskModel.create/save()        → workflow state
  - ChatSessionModel.create/save()      → messages
  - AgentRunModel.create/save()         → scan results
  - OpportunityModel.create()           → findings
