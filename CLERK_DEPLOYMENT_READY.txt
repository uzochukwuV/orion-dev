================================================================================
                        AUTHENTICATION FLOW
================================================================================

SIGNUP:
  User → /signup form → Clerk API → Webhook → MongoDB → /dashboard

LOGIN:
  User → /login form → Clerk API → JWT token → /dashboard

API CALL:
  Frontend sends: Authorization: Bearer {JWT}
  Backend: Verifies with Clerk
  Backend: Looks up user_id
  Backend: Returns only that user's data

LOGOUT:
  User clicks logout → Clerk clears session → Redirected to /login

PROTECTED ROUTE:
  User clicks /leads → ProtectedRoute checks isSignedIn
  Not signed in? Redirect to /login
  Signed in? Show /leads page
